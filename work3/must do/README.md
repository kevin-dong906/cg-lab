Phong光照模型实验报告

实验基本信息
1.1 实验名称：用Taichi做Phong光照模型，实现可交互的渲染效果

1.2 实验目的：搞懂局部光照的基本原理，学会三维向量的相关运算，最后能用Taichi做出可以交互的渲染效果。

实验目标
2.1 理论层面：弄明白环境光、漫反射和镜面高光到底是什么，能分清它们之间的区别，不用死记硬背，理解就好。

2.2 数学基础：熟练用好转三维向量的相关运算，比如法向量怎么算、光线往哪个方向走、视线方向和反射向量怎么求。

2.3 实际操作：会用Taichi做交互式渲染，能通过界面上的控件，实时调整材质参数，直观看到这些参数对最终渲染效果的影响。

实验原理
3.1 Phong光照模型的核心很简单，就是把物体表面反射的光，分成三个部分来计算，最后加起来就是我们看到的像素颜色，公式是$$I = I_{ambient} + I_{diffuse} + I_{specular}$$。

3.2 三个光分量的计算方法，其实也不复杂

3.2.1 环境光：就是场景里到处都有的背景光，不用特意照射，物体就能被照亮一点，公式是$$I_{ambient} = K_a \times C_{light} \times C_{object}$$。

3.2.2 漫反射：就像粗糙的墙面，光照射上去会往四面八方散开，亮度和光线照射的角度有关，遵循Lambert定律，公式是$$I_{diffuse} = K_d \times \max(0, \mathbf{N} \cdot \mathbf{L}) \times C_{light} \times C_{object}$$。

3.2.3 镜面高光：类似镜子反射的光，比较亮、比较集中，和我们看的角度有关，公式是$$I_{specular} = K_s \times \max(0, \mathbf{R} \cdot \mathbf{V})^n \times C_{light}$$。

3.3 几个关键的向量，简单记一下就好：$$\mathbf{N}$$是物体表面的法向量，$$\mathbf{L}$$是光线指向物体的方向，$$\mathbf{V}$$是我们眼睛看向物体的方向，$$\mathbf{R}$$是光线反射的方向，$$n$$是高光指数，决定高光的集中程度。

实验任务与步骤
4.1 先搭建一个简单的三维场景

4.1.1 不用找现成的模型文件，自己用代码定义两个几何体就行：左边放一个红色的球，球心在(-1.2,-0.2,0)，半径1.2，颜色选深红色(0.8,0.1,0.1)；右边放一个紫色的圆锥，顶点在(1.2,1.2,0)，底面在y=-1.4的位置，半径1.2，颜色选紫色(0.6,0.2,0.8)。

4.1.2 摄像机和光源的位置固定好，摄像机放在(0,0,5)，点光源放在(2,3,4)，光源是纯白色的(1.0,1.0,1.0)，背景色选深青色，看着舒服一点。

4.2 实现光线求交和深度测试

4.2.1 屏幕上的每一个像素，都相当于发射一条光线，我们要计算这条光线会不会撞到球体和圆锥，撞到的话，记录一下距离（用t表示）。

4.2.2 这里有个关键，要是光线同时撞到两个物体，就选离摄像机最近的那个交点（也就是t最小的那个正数），这样才能保证遮挡关系是对的，比如球挡住圆锥的一部分，就不会出现圆锥穿到球里面的情况，还要算出这个交点的坐标和法向量。

4.3 编写Phong着色器

4.3.1 找到光线撞到物体的交点后，算出这个点的$$\mathbf{L}$$、$$\mathbf{V}$$、$$\mathbf{R}$$三个向量。

4.3.2 按照之前的公式，分别算出环境光、漫反射、镜面高光这三个部分的颜色，加起来就是这个像素最终的颜色。

4.4 做一个可交互的UI面板

4.4.1 用Taichi自带的ti.ui.Window创建一个窗口，在窗口里加4个滑动条，方便我们调节参数。

4.4.2 滑动条分别绑定四个参数：Ka（环境光系数），范围0.0到1.0，默认0.2；Kd（漫反射系数），范围0.0到1.0，默认0.7；Ks（镜面高光系数），范围0.0到1.0，默认0.5；Shininess（高光指数），范围1.0到128.0，默认32.0。

实验实现
5.1 先初始化Taichi环境，把窗口的分辨率、存储像素的字段，还有要调节的交互参数，都定义好。

5.2 写两个小函数，一个用来归一化向量，一个用来计算反射向量，再写两个函数，分别判断光线有没有撞到球体和圆锥。

5.3 编写渲染的核心代码，实现光线投射、深度测试，还有Phong光照的计算，最后给每个像素上色。

5.4 写主函数，创建窗口，把参数初始化好，把滑动条和参数绑定起来，实现实时渲染，调节滑动条就能看到效果变化。

实验结果与分析
6.1 实验结果还不错，成功渲染出了红色的球和紫色的圆锥，遮挡关系也对，没有出现穿模的情况；通过界面上的滑动条，能实时调节参数，光照效果也会跟着变，很直观。

6.2 简单分析一下：调大Ka，环境光就变亮，整个物体都会更亮一点；调大Kd，漫反射的效果更明显，物体表面的明暗过渡会更自然；调大Ks和Shininess，镜面高光会更亮、更集中，有点像镜子反光的感觉。

实验总结
7.1 这次实验算是成功了，用Taichi实现了Phong光照模型的渲染，也彻底搞懂了局部光照的核心原理，还有三维向量的运算方法，不再是死记硬背公式了。

7.2 也熟练用了Taichi的Kernel和UI模块，做出了可交互的渲染效果，清楚了每个材质参数对光照效果的影响，实践能力也提升了不少。

7.3 实验中也遇到了点小问题，比如圆锥的光线求交逻辑有点复杂，一开始总出错，后来慢慢调试，调整了局部坐标系的转换和交点的验证，就解决了；还有深度测试，一开始没做好，出现了遮挡错误，后来完善了逻辑，就没问题了。
